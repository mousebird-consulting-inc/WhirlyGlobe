MaplyPagingDelegate

@protocol MaplyPagingDelegate

@brief The paging delegate is used by the paging layer to load tiles. @details The Maply Paging Delegate is used by the MaplyQuadPagingLayer to do feature (e.g. not image) paging. You set up an object that implements this protocol and talks to the paging layer. This is how you load things like vector tiles. @details It’s up to you to do the actual loading of tiles and turn the data into Maply features. Once you do that, the paging layer will handle the rest. - see: MaplyQuadPagingLayer

  • -startFetchForTile:forLayer:

    @brief Start fetching data for the given tile. @details The paging layer calls this method to let you know you should start fetching data for a given tile. This will not be called on the main thread so be prepared for that. @details You should immediately do an async call to your own loading logic and then merge in your results. If you do your loading calls in line you’ll slow down the loading thread. After you’re done you MUST call tileDidLoad in the layer, even for failures. @details Once you’ve loaded your data, you need to create the corresponding Maply objects in the view controller and pass them back to the paging layer for tracking. - parameter: tileID The tile the system wants you to start loading. - parameter: layer The quad paging layer you’ll hand the data over to when it’s loaded.

Bigger tiles may not work correctly. @details This is intended to allow 3D loading calculations to work better with divergent height data. - parameter: ll Lower left corner.
